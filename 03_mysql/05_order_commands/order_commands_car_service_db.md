# Порядок команд у SQL-запитах / SQL clause order — `car_service_db`

У навчальних матеріалах наведено **порядок запису** фраз у запиті. Його варто дотримуватися, щоб синтаксис був коректним. Нижче — той самий порядок, що на слайді, з поясненням ролі кожної фрази та прикладами на базі **`car_service_db`** (дамп `01_database_mysql/car_service_db.sql.gz`).

**Файл із запитами:** `05_order_commands/car_service_order_examples.sql` (з кореня репозиторію).

```bash
gunzip -c 01_database_mysql/car_service_db.sql.gz | mysql -u ... -p ... car_service_db
mysql -u ... -p ... car_service_db < 05_order_commands/car_service_order_examples.sql
```

**Важливо:** у движка логіка **обчислення** (спочатку `FROM`/`JOIN`, потім `WHERE`, потім `GROUP BY`/`HAVING`, у кінці `SELECT`-вирази, сортування й обрізання рядків) відрізняється від порядку **написання** — але **писати** треба саме в стандартному порядку нижче.

---

## Порядок команд (як у зображенні)

| № | Команда / фраза | Роль (коротко) |
|---|------------------|----------------|
| 1 | **`SELECT`** | Які стовпці або агрегати показати в результаті |
| 2 | **`FROM`** | Звідки брати рядки (базова таблиця) |
| 3 | **`JOIN`** | З’єднати інші таблиці за умовою (`INNER` / `LEFT` тощо) |
| 4 | **`WHERE`** | Відфільтрувати **рядки до** групування |
| 5 | **`GROUP BY`** | Розбити рядки на групи (для агрегатів) |
| 6 | **`HAVING`** | Відфільтрувати **групи** після агрегації |
| 7 | **`ORDER BY`** | Відсортувати результат |
| 8 | **`LIMIT`** | Обмежити кількість рядків у відповіді |
| 9 | **`LIMIT` … `OFFSET` …** | Пропустити перші *N* рядків, потім взяти наступні (пагінація) |

Не кожен запит містить усі фрази: наприклад, без групування не буде **`GROUP BY`** / **`HAVING`**. У файлі `.sql` є **повний** приклад із усіма пунктами зі списку та коротші варіанти.

## Зв’язок ізі схемою `car_service_db`

Приклади використовують **`work_orders`**, **`vehicles`**, **`customers`**, **`car_brands`**: з’єднання «замовлення → авто → клієнт», агрегати по клієнту або по статусу замовлення.

---

## Вправи (відповідність блокам у `car_service_order_examples.sql`)

1. **Повний ланцюжок** — `SELECT` … `FROM` … `JOIN` … `WHERE` … `GROUP BY` … `HAVING` … `ORDER BY` … `LIMIT` … `OFFSET` (перший «вікно» результатів: `LIMIT 10 OFFSET 0`).
2. **Той самий ланцюжок + пагінація** — `LIMIT 5 OFFSET 10` (наступні рядки після пропуску).
3. **Лише `LIMIT` без `OFFSET`** — після `ORDER BY` на `work_orders`.
4. **Без `GROUP BY` / `HAVING`** — `JOIN`, `WHERE`, `ORDER BY`, `LIMIT`.
5. **Альтернативний синтаксис** — `LIMIT offset, row_count` (еквівалент `LIMIT row_count OFFSET offset`).

Якщо другий запит із `OFFSET 10` повертає мало рядків — збільте діапазони `id` у `WHERE` або зменште `OFFSET`.
